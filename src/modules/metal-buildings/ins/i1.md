# What IdeaRoom Probably Does Internally

Their buildings are probably NOT fully modeled manually per configuration.

That would be impossible to scale.

Instead, they likely use:

# Parametric Generation

Meaning:

The building is assembled dynamically from:

* width
* length
* height
* roof style
* panel type
* lean-tos
* trim
* colors
* openings
* doors/windows

Example:

```js
generateBuilding({
  width: 30,
  length: 40,
  height: 12,
  roofType: "vertical",
  leanToLeft: true
})
```

Then the engine:

* creates meshes
* scales parts
* duplicates trusses
* positions panels
* recalculates materials

That’s how configurators work.

# Why React Three Fiber Matters

This is the important part.

DON’T build raw Three.js manually unless:

* you’re a graphics engineer
* you want maximum low-level control

Use:

* React Three Fiber (R3F)

Why?
Because:

* component-based architecture
* reusable building parts
* React state integration
* easier scene management
* cleaner scaling

Example:

```jsx
<Building>
  <Roof />
  <Wall />
  <LeanTo />
  <Door />
</Building>
```

This is PERFECT for your modular system.

## 2. Pricing Architecture

This is harder than rendering.

Because:

* taxes

This becomes enterprise-grade complexity VERY quickly.

---

## 3. Performance

Bad configurators die because:

* too many draw calls
* unoptimized meshes
* rerender loops
* memory leaks
* giant textures

You need:

* instancing
* geometry reuse
* LODs later
* lazy loading
* compressed textures
